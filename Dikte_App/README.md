# دیکته — نسخه حرفه‌ای (Dikte Professional)

یک اپلیکیشن وب حرفه‌ای برای تمرین دیکته فارسی با قابلیت‌های پیشرفته و رابط کاربری مدرن.

## 🌟 ویژگی‌ها

### ✨ ویژگی‌های اصلی
- **داشبورد هوشمند**: نمایش آمار و فعالیت‌های اخیر
- **ایجاد دیکته**: امکان ایجاد دیکته‌های جدید با دسته‌بندی و سطح دشواری
- **تمرین تعاملی**: سیستم تمرین کلمه به کلمه با بازخورد فوری
- **تاریخچه کامل**: ثبت و نمایش تمام تمرینات انجام شده
- **پشتیبانی از صدا**: تبدیل متن به گفتار با تنظیمات قابل تغییر

### 🎨 رابط کاربری
- **طراحی مدرن**: استفاده از Tailwind CSS و انیمیشن‌های روان
- **حالت تاریک**: پشتیبانی کامل از تم تاریک و روشن
- **واکنش‌گرا**: سازگار با تمام دستگاه‌ها (موبایل، تبلت، دسکتاپ)
- **RTL**: پشتیبانی کامل از زبان فارسی و چینش راست به چپ
- **دسترسی‌پذیری**: رعایت استانداردهای WCAG

### 🔧 قابلیت‌های فنی
- **PWA**: قابلیت نصب روی دستگاه‌ها
- **آفلاین**: کارکرد کامل بدون اینترنت
- **ذخیره‌سازی پیشرفته**: استفاده از IndexedDB و localStorage
- **Service Worker**: کش هوشمند و همگام‌سازی پس‌زمینه
- **مدولار**: معماری ماژولار و قابل توسعه

### 📊 آمار و گزارش‌گیری
- **آمار تمرین**: درصد موفقیت، زمان مطالعه، تعداد کلمات
- **فیلترهای پیشرفته**: فیلتر بر اساس تاریخ، دسته‌بندی، سطح دشواری
- **نمودار پیشرفت**: نمایش روند بهبود مهارت‌ها
- **گزارش‌های تفصیلی**: تحلیل عملکرد کاربر

## 🚀 نصب و راه‌اندازی

### پیش‌نیازها
- Node.js (نسخه 14 یا بالاتر)
- npm یا yarn
- مرورگر مدرن با پشتیبانی از ES6+

### نصب محلی

1. **کلون کردن پروژه**
```bash
git clone https://github.com/your-username/dikte-app.git
cd dikte-app
```

2. **نصب وابستگی‌ها**
```bash
npm install
```

3. **اجرای سرور توسعه**
```bash
npm run dev
```

4. **باز کردن در مرورگر**
```
http://localhost:3000
```

### ساخت برای تولید

```bash
npm run build
```

فایل‌های آماده برای تولید در پوشه `dist` قرار می‌گیرند.

## 🌐 استقرار (Deployment)

### 🚀 استقرار سریع (توصیه شده)
1. **فایل `deploy.bat` را دوبار کلیک کنید**
2. دستورالعمل‌های نمایش داده شده را دنبال کنید
3. اپلیکیشن شما در چند دقیقه آنلاین خواهد شد!

### 📖 راهنمای کامل
برای دستورالعمل‌های تفصیلی، فایل `deploy.md` را مطالعه کنید.

### 🆓 گزینه‌های رایگان میزبانی
- **Netlify** (توصیه شده) - بهترین برای سایت‌های استاتیک
- **Vercel** - عالی برای اپلیکیشن‌های وب مدرن
- **GitHub Pages** - رایگان و قابل اعتماد
- **Firebase Hosting** - سرویس میزبانی گوگل

همه گزینه‌ها کاملاً رایگان هستند و شامل:
- رمزگذاری HTTPS
- CDN جهانی
- استقرار خودکار
- دامنه سفارشی (اختیاری)

### استقرار روی Netlify

1. **اتصال به GitHub**
   - پروژه را در GitHub آپلود کنید
   - در Netlify روی "New site from Git" کلیک کنید
   - مخزن GitHub را انتخاب کنید

2. **تنظیمات ساخت**
   - Build command: (خالی بگذارید)
   - Publish directory: `.` (پوشه فعلی)

3. **دامنه سفارشی**
   - پس از استقرار، روی "Site settings" کلیک کنید
   - "Domain management" را انتخاب کنید
   - نام سایت را تغییر دهید

### استقرار روی Vercel

1. **نصب Vercel CLI**
```bash
npm i -g vercel
```

2. **استقرار**
```bash
vercel
```

### استقرار روی سرور سنتی

1. **ساخت پروژه**
```bash
npm run build
```

2. **آپلود فایل‌ها**
   - تمام فایل‌های پوشه `dist` را روی سرور آپلود کنید
   - مطمئن شوید که `index.html` در ریشه قرار دارد

3. **تنظیم سرور**
   - برای Apache، فایل `.htaccess` را اضافه کنید
   - برای Nginx، تنظیمات rewrite را پیکربندی کنید

## 📁 ساختار پروژه

```
dikte-app/
├── index.html                 # فایل اصلی HTML
├── manifest.json             # فایل PWA manifest
├── sw.js                     # Service Worker
├── assets/
│   ├── css/
│   │   └── styles.css        # استایل‌های سفارشی
│   ├── js/
│   │   ├── app.js           # فایل اصلی اپلیکیشن
│   │   ├── storage-manager.js # مدیریت ذخیره‌سازی
│   │   ├── ui-manager.js     # مدیریت رابط کاربری
│   │   ├── dikte-manager.js  # مدیریت دیکته‌ها
│   │   ├── practice-manager.js # مدیریت تمرین
│   │   └── audio-manager.js  # مدیریت صدا
│   └── icons/               # آیکون‌های PWA
├── package.json
└── README.md
```

## 🔧 پیکربندی

### تنظیمات PWA
فایل `manifest.json` را برای تغییر نام، آیکون‌ها و تنظیمات اپلیکیشن ویرایش کنید.

### تنظیمات Service Worker
فایل `sw.js` را برای تغییر استراتژی کش و تنظیمات آفلاین ویرایش کنید.

### تنظیمات ذخیره‌سازی
در `storage-manager.js` می‌توانید تنظیمات IndexedDB را تغییر دهید.
